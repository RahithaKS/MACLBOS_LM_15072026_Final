Solution for the 3 broken builders
Same fix pattern for all three:

Step 1 — Detect is_multi_year

is_multi_year = 'year' in _group_cols

Checked from group_by_clause (same technique as is_multi_month).

Step 2 — _build_budget_per_avg_capacity_entity_sql
Add a new is_multi_year path (alongside the existing is_multi_month path). In this path:

Window function: PARTITION BY year, {entity_col} ORDER BY month::int so each year's running sum stays isolated
JOIN: r.{entity_col} = o.{entity_col} AND r.year = o.year AND r.month::int = o.month_count
Revenue and capacity CTEs: both use original year IN (2025,2026) AND month = 3 (no YTD expansion needed — same month, different years)
SELECT: includes both r.year and r.{entity_col}
Step 3 — _build_budget_per_capacity_sql
Fix the JOIN from r.region_entity = c.region_entity to also include year when is_multi_year:

r.region_entity = c.region_entity AND r.year = c.year
SELECT: add r.year to output columns
Step 4 — _build_budget_per_avg_capacity_ww_sql
Same fix as entity builder's is_multi_year path (WW builder already has the first-col-only JOIN bug too — fix both at once).
